## 2. 实验1: 反思学习效果评估

### 2.1 实验设计

**目标**: 验证 Agent 能否从用户纠正中提取有效规则，并在后续对话中正确应用。

**流程**:
1. **基线测试**: 在无反思能力情况下回答问题
2. **反思学习**: 触发10次纠正场景，提取规则
3. **规则生效验证**: 重测相同问题，验证规则是否被查询
4. **持久化验证**: 检查规则是否正确保存到知识图谱

**数据规模**:
- 30个学术实体
- 15条初始规则
- 10个预设纠正场景
- 40个测试问答对

### 2.2 核心结果

| 指标 | 结果 | 说明 |
|------|------|------|
| **规则提取成功率** | **100%** (10/10) | 所有纠正场景均成功提取规则 |
| **规则命中次数** | **9次** | 规则被正确查询到的次数 |
| **持久化规则数** | **13条** | 成功保存到知识图谱的规则 |

### 2.3 规则提取示例

| 纠正场景 | 提取的规则（摘要） |
|----------|------------|
| Transformer自注意力序列长度 | 必须严格比对输入与输出张量的维度变换细节 |
| BERT不适用于生成任务 | 评估模型适用性时需检查信息流向约束 |
| GPT单向因果注意力 | 区分编码器与解码器的信息可见范围 |

### 2.4 技术亮点

1. **结构化规则提取**: LLM能够将用户纠正转化为可执行的推理规则
2. **实体关联**: 规则自动关联到相关实体，支持后续精确检索
3. **持久化可靠**: 规则以JSON格式存储，重启后自动加载

## 3. 实验2: 混合检索消融实验

### 3.1 实验设计

**目标**: 量化知识图谱（KG）和向量检索各自对系统性能的贡献。

**配置对比**:
| 配置 | KG检索 | 向量检索 | 说明 |
|------|--------|----------|------|
| Baseline | ✗ | ✗ | 仅工作记忆 |
| Vector-only | ✗ | ✓ | 纯向量RAG |
| KG-only | ✓ | ✗ | 纯知识图谱 |
| Hybrid | ✓ | ✓ | 完整系统 |

### 3.2 检索效率对比

| 检索类型 | 平均延迟 | 说明 |
|----------|----------|------|
| **KG检索** | **1ms** | NetworkX内存图查询，极快 |
| **向量检索** | **31ms** | ChromaDB向量相似度计算 |
| **上下文组装** | **11ms** | Prompt拼接 |

### 3.3 技术亮点

1. **KG检索极快**: 内存图查询仅需1ms，适合实时对话
2. **向量检索高效**: 31ms完成相似度计算，对整体延迟影响小
3. **混合架构**: 两种检索可并行执行，不影响响应时间

## 4. 实验3: 系统性能基准测试

### 4.1 实验设计

**目标**: 测量系统各组件的耗时分布，识别性能瓶颈。

### 4.2 分段耗时分析

| 阶段 | 平均耗时 | 占比 |
|------|----------|------|
| 实体提取 | 641ms | 3.7% |
| KG检索 | 1ms | <0.1% |
| 向量检索 | 31ms | 0.2% |
| 上下文组装 | 11ms | <0.1% |
| LLM推理 | 16,460ms | 96% |

### 4.3 关键发现

1. **检索层高效**: KG+向量检索仅占 <1% 时间
2. **LLM为主导**: 96%时间用于LLM推理（这是预期行为）
3. **实体提取可优化**: 641ms可通过缓存优化

### 4.4 优化建议

| 优化点 | 方法 | 预期效果 |
|--------|------|----------|
| 实体提取 | 结果缓存 | 减少90%+调用 |
| LLM推理 | 流式输出 | 提升用户体验 |
| 向量检索 | ANN索引 | 支持更大规模 |

## 5. 简历素材

### 5.1 项目描述（推荐）

> **MemoAgent - 具备反思学习能力的智能对话系统**
>
> - 设计**三层记忆架构**（语义/情节/工作记忆），实现混合检索RAG系统
> - 实现反思学习机制：**规则提取成功率100%**，支持从用户纠正中自动沉淀推理规则
> - 检索效率优化：**KG检索1ms**、向量检索**31ms**，检索层耗时占比<1%
> - 技术栈：Python/FastAPI/LangChain/NetworkX/ChromaDB/React/TypeScript

### 5.2 量化指标汇总

| 维度 | 指标 | 数值 |
|------|------|------|
| **反思学习** | 规则提取成功率 | **100%** |
| | 规则命中验证 | 9次 |
| | 持久化规则数 | 13条 |
| **检索效率** | KG检索延迟 | **1ms** |
| | 向量检索延迟 | **31ms** |
| | 检索占比 | **<1%** |
| **系统架构** | 实体规模 | 30个 |
| | 规则规模 | 15条初始+13条学习 |
| | 测试场景 | 10个纠正+40个问答 |

### 5.3 技术亮点描述

**适用于"项目亮点"或"技术难点"部分**:

1. **反思学习机制**：设计Prompt模板引导LLM从用户纠正中提取结构化规则，规则自动关联实体并存入知识图谱，实现"从错误中学习"

2. **混合检索架构**：结合知识图谱（精确规则查询）与向量检索（语义相似匹配），两者延迟分别仅为1ms和31ms，检索层不影响对话响应速度

3. **三层记忆设计**：
   - 语义记忆（NetworkX）：存储实体关系和推理规则
   - 情节记忆（ChromaDB）：向量化历史对话，支持相似性检索
   - 工作记忆：滑动窗口管理当前会话上下文
